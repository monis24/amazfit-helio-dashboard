## Sleep stage mode-code derivation (empirical)

Reference sources only documented mode 4 (light) and 5 (deep); this project's own review flagged REM/Awake codes as unconfirmed. Derived here by summing per-mode minutes across all fetched days' `stage`/`odd_stage` arrays and cross-checking against each record's `slp.lt`/`slp.dp`/`slp.wk` fields and `ed - st` (total session minutes):

- mode 4: 663 total minutes across sampled days
- mode 5: 235 total minutes across sampled days
- mode 7: 66 total minutes across sampled days
- mode 8: 249 total minutes across sampled days

**Result:** mode 4 = Light, mode 5 = Deep, mode 7 = Awake, mode 8 = REM. Confirmed by exact-sum arithmetic in at least one sample (mode durations summed to `ed - st`, and mode 4/5/7 minutes matched `slp.lt`/`slp.dp`/`slp.wk` exactly) — not a guess. Re-verify if a differently-configured account or firmware version produces additional mode codes.

**HR blob encoding confirmed:** `data_hr` decodes to 1440 raw uint8 bytes (one per minute of day), not int16 pairs as the reference research suggested. 254 is the dominant "no reading" sentinel value observed live.
